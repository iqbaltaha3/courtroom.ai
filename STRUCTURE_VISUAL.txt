KEY CONCEPT: ".." means "go up one directory level"

src/
├── agents/          ← if you're here
│   └── case_analyst.py
│
└── services/        ← ".." gets you here (parent level)
    └── llm_service.py

WHY THIS MATTERS FOR INTERVIEWS:

1. SCALABILITY    - Add new agents? Just add to src/agents/
2. TESTABILITY    - Test agents independently without UI
3. MAINTAINABILITY - Change LLM? Only update llm_service.py
4. READABILITY    - New engineer sees structure instantly
5. PROFESSIONALISM - Looks like experienced team wrote it
